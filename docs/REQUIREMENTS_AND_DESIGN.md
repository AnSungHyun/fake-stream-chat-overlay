# Fake Stream Chat Overlay 요구사항 및 설계 분석

## 1. 프로젝트 개요

이 프로젝트는 게임 유튜버, 스트리머, 초보 크리에이터가 영상 촬영 중 화면 위에 자연스러운 멀티 플랫폼 채팅 흐름과 도네이션 알림을 띄울 수 있도록 만든 Windows용 투명 오버레이 프로그램이다.

초기 문제의식은 다음과 같다.

- 막 시작한 크리에이터는 실제 채팅 반응이 부족해 영상 텐션이 떨어질 수 있다.
- 게임 유튜버 영상에서 흔히 보이는 YouTube, CHZZK, SOOP 스타일의 채팅 흐름을 촬영 보조용으로 재현하고 싶다.
- 단순한 고정 텍스트가 아니라 시청자 수, 닉네임 색상, 플랫폼 뱃지, 채팅 간격, 도네이션 알림 등이 자연스럽게 움직여야 한다.
- 이후에는 현재 화면 스크린샷을 분석해 LLM이 상황에 맞는 채팅 키워드와 문장을 생성하는 방향으로 확장할 수 있다.

## 2. 핵심 사용자

- 게임 유튜브 영상을 막 시작하는 크리에이터
- 실제 라이브 채팅 없이도 라이브 같은 영상 분위기를 만들고 싶은 사용자
- OBS, 화면 녹화, 편집용 소스로 투명 오버레이를 활용하고 싶은 사용자
- 촬영 중 채팅 분위기, 시청자 수, 후원 알림을 빠르게 조절하고 싶은 사용자

## 3. 주요 요구사항

### 3.1 Windows 투명 채팅 오버레이

- Windows 화면 위에 투명한 채팅창이 표시되어야 한다.
- 채팅 메시지가 아래에서 위로 자연스럽게 누적되어야 한다.
- 오버레이 창은 항상 위에 표시될 수 있어야 한다.
- 필요 시 클릭 통과 모드를 켜서 게임 조작이나 다른 창 클릭을 방해하지 않아야 한다.
- 오버레이 창은 프레임 없는 투명 창이어야 한다.

### 3.2 컨트롤 창

- 실행 후 별도의 컨트롤 창에서 옵션을 조절할 수 있어야 한다.
- 컨트롤 창에서 변경한 설정은 오버레이에 즉시 반영되어야 한다.
- 컨트롤 창을 닫으면 채팅 오버레이와 도네이션 오버레이도 함께 종료되어야 한다.

### 3.3 채팅 흐름 설정

- 채팅 인원 수를 조절할 수 있어야 한다.
- 채팅 생성 간격을 조절할 수 있어야 한다.
- 채팅 등장 애니메이션 속도를 조절할 수 있어야 한다.
- 화면에 유지할 최대 채팅 줄 수를 조절할 수 있어야 한다.
- 채팅 인원 수에 따라 가상 닉네임 풀이 생성되고, 같은 닉네임이 반복 등장해 실제 채팅처럼 보여야 한다.

### 3.4 채팅 표시 옵션

- 채팅창 투명도를 조절할 수 있어야 한다.
- 글자 크기를 조절할 수 있어야 한다.
- 아이디 색상을 랜덤으로 지정할 수 있어야 한다.
- 컴팩트 모드를 지원해야 한다.
- 플랫폼 뱃지 표시 여부를 조절할 수 있어야 한다.
- YouTube, CHZZK, SOOP 플랫폼을 각각 켜고 끌 수 있어야 한다.

### 3.5 채팅 내용 생성

- 감탄사, 질문, 게임 플레이 반응, 잡담 패턴을 선택할 수 있어야 한다.
- 화면 키워드를 입력하면 해당 키워드가 포함된 채팅이 생성되어야 한다.
- 스트리머 호칭을 설정할 수 있어야 한다.
- 예시 패턴:
  - "와 이걸 사네"
  - "방금 뭐임?"
  - "{streamer} 이거 무슨 캐릭이에요?"
  - "{keyword} 타이밍 좋았다"

### 3.6 시청자 수

- LIVE / watching 헤더가 표시되어야 한다.
- 시청자 수가 랜덤 워크 방식으로 조금씩 증가하거나 감소해야 한다.
- 헤더는 채팅 메시지 수 변화와 관계없이 상단에 고정되어야 한다.
- 시청자 수 표시 여부와 변동 폭을 조절할 수 있어야 한다.

### 3.7 오버레이 크기 및 위치

- 채팅 오버레이는 화면에서 드래그해 위치를 옮길 수 있어야 한다.
- 가장자리와 모서리 리사이즈 손잡이를 통해 창 크기를 조절할 수 있어야 한다.
- 컨트롤 창에서 오버레이 위치를 기본 위치로 되돌릴 수 있어야 한다.

### 3.8 도네이션 오버레이

- 도네이션 알림은 채팅창과 별도의 투명 오버레이 창으로 분리되어야 한다.
- 기본 위치는 화면 좌측 위 근처여야 한다.
- 도네이션 알림이 뜰 때 동전 느낌의 "짜라랑" 사운드가 재생되어야 한다.
- 도네이션 알림은 평소에는 투명하고, 표시 시에만 카드 형태로 나타나야 한다.
- 도네이션 알림은 다음 옵션을 지원해야 한다.
  - 사용 여부
  - 자동 표시 간격
  - 표시 시간
  - 알림 투명도
  - 최소 후원 금액
  - 최대 후원 금액
  - 테스트 표시
  - 위치 초기화

## 4. 기술 선택

### 선택 기술: Electron + HTML/CSS/JavaScript

Electron을 선택한 이유:

- Windows 투명 창 구현이 쉽다.
- 항상 위 표시, 프레임 없는 창, 창 위치/크기 제어가 가능하다.
- 별도 네이티브 UI 프레임워크 없이 HTML/CSS로 빠르게 UI를 만들 수 있다.
- `npm install`, `npm start`, `run.bat` 방식으로 실행 진입 장벽이 낮다.
- 이후 패키징, 자동 업데이트, OBS용 웹 버전 확장이 가능하다.

## 6. 창 구성

### 6.1 컨트롤 창

- 사용자가 설정을 변경하는 메인 창이다.
- 설정 변경 시 IPC를 통해 채팅 오버레이와 도네이션 오버레이에 전달한다.
- 컨트롤 창이 닫히면 앱 전체가 종료된다.

### 6.2 채팅 오버레이 창

- 투명한 프레임리스 창이다.
- 기본적으로 화면 오른쪽 쪽에 위치한다.
- LIVE / watching 헤더와 채팅 리스트를 표시한다.
- 창 내부 모서리/가장자리에서 리사이즈할 수 있다.

### 6.3 도네이션 오버레이 창

- 채팅 오버레이와 별도의 투명 창이다.
- 기본 위치는 화면 좌측 위 근처다.
- 도네이션 발생 시 카드가 나타나고 사운드를 재생한다.
- 컨트롤 창에서 수동 테스트와 위치 초기화가 가능하다.

## 7. IPC 이벤트 설계

- `settings:get-defaults`: 기본 설정 조회
- `settings:update`: 컨트롤 창에서 설정 변경 사항 전파
- `overlay:get-bounds`: 채팅 오버레이 현재 위치/크기 조회
- `overlay:set-bounds`: 채팅 오버레이 위치/크기 변경
- `overlay:reset`: 채팅 메시지 비우기
- `overlay:sample-burst`: 샘플 채팅 여러 개 즉시 생성
- `overlay:toggle-visibility`: 채팅 오버레이 표시/숨김
- `overlay:recenter`: 채팅 오버레이 기본 위치로 이동
- `donation:get-bounds`: 도네이션 창 위치/크기 조회
- `donation:set-bounds`: 도네이션 창 위치/크기 변경
- `donation:manual`: 도네이션 알림 수동 표시
- `donation:recenter`: 도네이션 창 좌측 위 기본 위치로 이동
