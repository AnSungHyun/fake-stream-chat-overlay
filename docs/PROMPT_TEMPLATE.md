# 동일 결과물 생성을 위한 프롬프트 템플릿

이 문서는 AI 코딩 에이전트에게 현재 프로젝트와 같거나 더 나은 프로그램을 만들도록 요청할 때 사용할 수 있는 프롬프트 템플릿이다.

## 1. 짧은 요청 프롬프트

```text
Windows에서 실행되는 Electron 기반 투명 스트리밍 채팅 오버레이 앱을 만들어줘.

목표는 게임 유튜버/스트리머가 영상 촬영 중 실제 라이브처럼 보이는 가짜 멀티 플랫폼 채팅창과 도네이션 알림을 띄우는 것이다.

작업 디렉토리 안에 새 프로젝트를 만들고, 실행은 npm install 후 npm start 또는 run.bat 더블클릭으로 가능하게 해줘.

필수 기능:
- 투명한 채팅 오버레이 창
- 별도 컨트롤 창
- 채팅 오버레이는 항상 위 표시, 클릭 통과, 드래그 이동, 리사이즈 지원
- LIVE / watching 시청자 수 헤더는 채팅 줄 수와 무관하게 상단 고정
- 시청자 수는 실시간으로 조금씩 증가/감소
- 채팅 인원, 채팅 간격, 애니메이션 속도, 최대 표시 줄 조절
- YouTube, CHZZK, SOOP 플랫폼 뱃지 켜기/끄기
- 닉네임 랜덤 색상
- 채팅창 투명도와 글자 크기 조절
- 감탄사, 질문, 게임 반응, 잡담 패턴으로 자동 채팅 생성
- 화면 키워드와 스트리머 호칭을 입력하면 채팅 문장에 반영
- 도네이션 알림은 채팅창과 별도의 투명 오버레이 창으로 분리
- 도네이션 기본 위치는 화면 좌측 위 근처
- 도네이션은 동전 소리와 함께 카드 형태로 표시
- 도네이션 사용 여부, 간격, 표시 시간, 투명도, 최소/최대 금액, 수동 테스트, 위치 초기화 지원
- 컨트롤 창을 닫으면 모든 오버레이 창도 같이 종료

구현 후 README와 요구사항/설계 문서, 재현용 프롬프트 템플릿 문서도 작성해줘.
마지막에 JavaScript 문법 검사와 npm start 실행 확인까지 해줘.
```

## 2. 권장 상세 프롬프트

```text
작업 디렉토리: C:\HyunDev\
프로젝트 이름: fake-stream-chat-overlay

Windows용 데스크톱 앱을 만들어줘. 기술은 Electron + HTML/CSS/JavaScript를 사용해줘. TypeScript나 React 없이도 괜찮고, 실행과 수정이 쉬운 구조를 우선해줘.

앱의 목적:
게임 유튜버/스트리머가 영상 촬영 중 화면 위에 실제 라이브처럼 보이는 가짜 멀티 플랫폼 채팅창과 도네이션 알림을 띄우는 프로그램.

창 구성:
1. 컨트롤 창
2. 채팅 오버레이 창
3. 도네이션 오버레이 창

컨트롤 창 요구사항:
- 앱 실행 시 함께 열린다.
- 모든 설정을 실시간으로 조절한다.
- 컨트롤 창을 닫으면 앱 전체가 종료된다.
- 다음 버튼을 제공한다.
  - 샘플 채팅 붓기
  - 채팅 비우기
  - 채팅 오버레이 위치 초기화
  - 도네이션 테스트
  - 도네이션 위치 초기화

채팅 오버레이 요구사항:
- 투명 배경, 프레임 없는 Electron BrowserWindow로 만든다.
- 항상 위 표시 옵션을 지원한다.
- 클릭 통과 옵션을 지원한다.
- 창 드래그 이동을 지원한다.
- 가장자리와 모서리 리사이즈 손잡이로 크기 조절을 지원한다.
- 기본 위치는 화면 오른쪽 근처로 잡는다.
- LIVE / watching 헤더는 채팅 리스트와 분리해 상단에 고정한다.
- 채팅 줄 수가 늘거나 줄어도 헤더 위치가 흔들리지 않게 한다.
- 채팅 메시지는 아래에서 위로 쌓이게 표시한다.
- 메시지 등장 애니메이션을 적용한다.
- 채팅창 투명도, 글자 크기, 최대 줄 수, 컴팩트 모드 옵션을 지원한다.

채팅 생성 요구사항:
- 채팅 인원 수에 따라 가상 시청자 닉네임 풀을 만든다.
- 같은 닉네임이 반복 등장해 실제 채팅처럼 보이게 한다.
- 닉네임 색상은 랜덤 색상 옵션으로 제어한다.
- YouTube, CHZZK, SOOP 플랫폼 뱃지를 지원하고 각각 켜고 끌 수 있게 한다.
- 채팅 문장 패턴은 감탄사, 질문, 게임 플레이 반응, 잡담으로 나눈다.
- 각 패턴은 켜고 끌 수 있게 한다.
- 화면 키워드 입력값을 `{keyword}` 자리에 넣어 게임 상황 반응처럼 보이게 한다.
- 스트리머 호칭 입력값을 `{streamer}` 자리에 넣는다.

시청자 수 요구사항:
- LIVE / watching 헤더에 시청자 수를 표시한다.
- 시작 시청자 수와 변동 폭을 조절할 수 있게 한다.
- 일정 시간마다 랜덤 워크 방식으로 숫자를 증가/감소시킨다.
- 숫자 표시 폭을 고정하거나 tabular 숫자를 사용해 헤더가 흔들리지 않게 한다.

도네이션 오버레이 요구사항:
- 채팅 오버레이 내부가 아니라 별도의 투명 BrowserWindow로 만든다.
- 기본 위치는 화면 좌측 위 근처, 예: x=80, y=80.
- 프레임 없는 투명 창이고 항상 위 표시를 적용한다.
- 평소에는 카드가 보이지 않다가 도네이션 이벤트 때만 나타나게 한다.
- 동전 느낌의 짧은 "짜라랑" 사운드를 Web Audio API로 재생한다.
- 도네이션 카드에는 닉네임, 금액, 후원 메시지를 표시한다.
- 자동 도네이션 간격은 분 단위로 조절한다.
- 표시 시간, 투명도, 최소 금액, 최대 금액을 조절한다.
- 수동 테스트 버튼을 지원한다.
- 위치 초기화 버튼을 지원한다.

기술 구현 요구사항:
- Electron main process에서 세 창을 생성한다.
- preload.js를 통해 안전한 IPC API를 노출한다.
- contextIsolation 기본 보안 모델을 유지한다.
- 설정 변경은 컨트롤 창에서 IPC로 두 오버레이 창에 전달한다.
- 실행 스크립트는 `npm start`로 제공한다.
- Windows 사용자를 위해 `run.bat`도 제공한다.
- `.gitignore`를 만들어 `node_modules`, `.idea`, `dist`, `.env` 등을 제외한다.

문서 요구사항:
- README.md에 실행 방법과 기능 요약을 작성한다.
- docs/REQUIREMENTS_AND_DESIGN.md에 요구사항, 설계 분석, 구현 구조, 향후 확장 아이디어를 작성한다.
- docs/PROMPT_TEMPLATE.md에 같은 결과물을 만들기 위한 짧은 프롬프트와 상세 프롬프트를 작성한다.

검증 요구사항:
- 다음 명령으로 JS 문법 검사를 수행한다.
  - node --check src/main.js
  - node --check src/preload.js
  - node --check src/overlay.js
  - node --check src/donation.js
  - node --check src/control.js
- `npm start`로 앱이 최소 몇 초 이상 정상 실행되는지 확인한다.
- 마지막에 변경 파일과 실행 방법을 요약한다.
```

## 3. 더 좋은 결과물을 위한 추가 요청 문장

아래 문장을 상세 프롬프트 끝에 추가하면 완성도가 더 좋아진다.

```text
UI는 촬영 도구처럼 조용하고 실용적으로 디자인해줘. 마케팅 랜딩 페이지처럼 만들지 말고, 바로 설정을 만질 수 있는 밀도 있는 도구 화면으로 구성해줘.

채팅 오버레이는 녹화 화면 위에서 너무 튀지 않게 반투명 어두운 패널을 사용하고, 글자 가독성은 유지해줘.

도네이션 오버레이는 채팅창과 완전히 독립적인 창이어야 하며, 좌측 위 기본 위치에서 자연스럽게 나타났다 사라지게 해줘.

나중에 OpenAI API를 붙여 현재 화면 스크린샷 기반 채팅 생성을 확장할 수 있도록 코드 구조를 너무 얽히지 않게 나눠줘.

이미 존재하는 사용자 파일을 함부로 삭제하지 말고, 필요한 파일만 최소 범위로 추가/수정해줘.
```
